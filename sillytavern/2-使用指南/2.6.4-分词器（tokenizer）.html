title: 分词器（Tokenizer）

分词器（Tokenizer）

分词器是一种将文本拆分为更小单元（称为 tokens）的工具。这些 tokens 可以是单个单词，甚至是单词的部分，如前缀、后缀或标点符号。一般来说，一个 token 通常对应于 3 到 4 个字符。

SillyTavern 提供了一个“最佳匹配”选项，它会根据所使用的 API 提供商，按照以下规则尝试匹配分词器。

文本补全 API（可覆盖）：

  1. NovelAI Clio: 使用 NerdStash 分词器。
  2. NovelAI Kayra: 使用 NerdStash v2 分词器。
  3. 文本补全 API: 使用 API 分词器（如果支持）或 Llama 分词器。
  4. KoboldAI Classic / AI Horde: 使用 Llama 分词器。
  5. Koboldcpp: 使用模型 API 分词器。

如果你遇到不准确的结果或想进行实验，可以为 SillyTavern 设置一个 覆盖分词器，以在请求生成时使用：

  1. 无：每个 token 估算为 ~3.3 个字符，四舍五入到最接近的整数。如果你的提示词（prompt）被截断，尝试此选项。此方法由 KoboldAI Lite 使用。
  2. Llama 分词器：用于 Llama 1/2 模型系列：Vicuna、Hermes、Airoboros 等。
  3. Llama 3 分词器：用于 Llama 3/3.1 模型。
  4. NerdStash 分词器：用于 NovelAI 的 Clio 模型。
  5. NerdStash v2 分词器：用于 NovelAI 的 Kayra 模型。
  6. Mistral V1 分词器：用于较旧的 Mistral 模型及其微调版本。
  7. Yi 分词器：用于 Yi 模型。
  8. Gemma 分词器：用于 Gemini/Gemma 模型。
  9. API 分词器：查询生成 API 直接获取模型的 token 数量。已知支持的后端：Text Generation WebUI（ooba）、Koboldcpp、TabbyAPI、Aphrodite API。

对话生成 API（不可覆盖）：

  1. OpenAI：依赖于模型的分词器，通过 tiktoken 实现。
  2. Claude：依赖于模型的分词器，通过 WebTokenizers 实现。
  3. OpenRouter：针对相应模型使用 Llama、Mistral、Gemma、Yi 分词器。
  4. Google AI Studio：使用 Gemma 分词器。
  5. Scale API：使用 GPT-4 分词器。
  6. AI21 API：使用 Jamba 分词器（需要一次性下载）。
  7. Cohere API：使用 Command-R 分词器（需要一次性下载）。
  8. MistralAI API：使用 Mistral V1 或 V3 分词器（需要一次性下载）。
  9. 回退分词器：使用 GPT-3.5 turbo 分词器。

附加分词器

以下分词器由于文件较大，未包含在默认安装中。第一次使用时需要下载：

  1. Qwen2 分词器。
  2. Command-R 分词器：用于 Cohere 源的对话生成。
  3. Mistral V3 (Nemo) 分词器：用于 MistralAI 源的对话生成（Nemo 和 Pixtral 模型）。

如果你不希望下载分词器，可以在 config.yaml 文件中启用“禁用下载选项”：enableDownloadableTokenizers，将其设置为 false 来禁用下载。

你也可以手动从 SillyTavern-Tokenizers 仓库下载分词器文件，将它们放到数据根目录下的 _cache 子目录中，默认路径为 ./data/_cache。如果该目录不存在，请创建它。之后，重新启动 SillyTavern 服务器以重新初始化分词器。

如果所需的分词器未缓存且禁用了下载功能，将使用回退分词器（Llama 3）来进行计数。

Token Padding（令牌填充）

适用范围：文本补全 API SillyTavern 始终会使用适配对话生成模型的匹配分词器，因此无需填充令牌。

除非 SillyTavern 使用了由远程后端 API 提供的分词器，否则所有的 token 数量估算都是基于所选择的 分词器 类型。

由于分词结果在接近模型定义的最大上下文大小时可能不准确，提示词（prompt）的某些部分可能会被裁剪或丢失，这可能会影响角色定义的一致性。

为了避免这种情况，SillyTavern 会将上下文大小的一部分分配为填充空间，以防添加过多的对话内容超出模型可以处理的范围。如果你发现即使选择了最匹配的分词器，某些部分的提示词（prompt）仍然被截断，可以调整填充量，以确保描述不会被截断。

你可以输入负值进行反向填充，从而分配比设定最大 token 数量更多的令牌。
