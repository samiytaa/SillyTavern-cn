2.6.2 上下文模板

上下文模板

适用于：文本补全 API

在 Chat 完成 API 中，等效的设置请使用 2.6.6 提示管理器。

通常，AI 模型需要你以特定的方式提供角色数据。SillyTavern 包含一系列为不同模型预设的转换规则，但你可以根据需要自定义它们。

在 “2.6.1 高级格式化” 面板中编辑这些设置。

故事字符串

此字段是用于聊天前角色数据的模板（内部称为故事字符串）。这是格式化角色卡片和指示模型的主要方式。

该模板支持 Handlebars 语法以及任何自定义文本注入或格式化。请参见此语言参考：https://handlebarsjs.com/guide/

我们向 Handlebars 评估器提供了以下参数（将它们包裹在双大括号中）：

  1. description - 角色的描述
  2. scenario - 角色的情景
  3. personality - 角色的个性
  4. system - 2.6.1 高级格式化 或角色的主要提示词覆盖（如果启用了“优先角色提示词”设置）
  5. persona - 选择的角色描述
  6. char - 角色的名称
  7. user - 选择的用户角色名称
  8. wiBefore 或 loreBefore - 设置为“角色定义之前”的激活世界信息条目组合
  9. wiAfter 或 loreAfter - 设置为“角色定义之后”的激活世界信息条目组合
  10. mesExamples - （可选）角色的示例对话，按照 instruct 格式处理，使用分隔符。将“示例消息行为”设置为“从不包含示例”以避免重复。

支持特殊的 {{trim}} 宏来移除它周围的任何换行符。如果你希望某部分文本不与前一行用换行分隔，可以使用它（空格不会被修剪）。

警告：如果模板中缺少上述任何参数，它们将根本不会出现在提示词中。

示例分隔符

用于作为块标题和示例对话块之间的分隔符。对话示例中的任何 <START> 标签都将替换为该字段的内容。

聊天开始

在渲染的故事字符串和示例对话块之后、但在第一个上下文消息之前插入为分隔符。

作为停止字符串的分隔符

将“示例分隔符”和“聊天开始”添加到停止字符串列表中。

如果模型倾向于虚构或泄露整个示例对话块并带有分隔符，这会有所帮助。

作为停止字符串的名称

将角色和用户角色的名称添加到停止字符串列表中。

建议保持开启，以防止模型模仿。

允许历史后指令

在提示词的最后包含历史后指令，按用户消息格式化。

历史后指令提示词应该在角色卡中定义，并且需要启用“优先角色指令”设置。

应谨慎使用，因为将指令放在上下文较低的位置可能导致较小模型的输出质量下降。

始终将角色名称添加到提示词

将角色的名称附加到提示词中，强制模型以该角色完成消息：
